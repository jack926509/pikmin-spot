SECTION 01專案目標與 KPI

開發一個 Telegram Bot,讓玩家不必手動 Google 地名,直接傳截圖就能拿到精確 GPS 座標。

首要 KPI

識別成功率 ≥ 90%(對於可被 Wikipedia 收錄的地標)。這是本版規格的最高優先級,所有架構決策都圍繞它展開。

次要目標

  • 月成本 < USD $1(假設 500 張圖/月)
  • 單次回應時間 < 12 秒(從收圖到回傳座標)
  • 三個必填 env var,皆免費取得,無需信用卡
  • 部署於 Zeabur,Long-polling 模式不需 webhook

SECTION 02識別率優化策略 ✦ 本版核心

過去單一 LLM + 單一 geocoder 的設計常在以下五個關卡崩盤,本版用對應策略一一破解:

弱點本版對策
LLM OCR 誤讀(如 Jangtsa 讀成 Jangtso)LLM 一次回 1~3 個候選名,逐一嘗試
單一 geocoder 對冷門地標漏接4 層級聯:Wikidata → Wikipedia → Nominatim → Photon
Geocoder 對拼寫敏感用候選名 × 原文(中/日/藏)交叉查詢
LLM 不熟 Pikmin Bloom UI 版位Prompt 內明確說明 UI 版位 + 兩個 few-shot 範例
知名地標被 geocoder 誤抓同名地點優先 Wikidata,語意 ID(如 Q3162338)永不混淆
為什麼 Wikidata 是 Game Changer:它對每個地標儲存 P625 座標屬性,精確且權威;支援多語搜尋(藏文 / 日文 / 中文 / 韓文都能搜);完全免費、無 key、無流量限制;對於 Pikmin Bloom 主打的觀光地標,覆蓋率極高(估 >85%)。

SECTION 05系統架構

┌──────────┐ photo ┌───────────┐ base64 ┌──────────────┐ │ Telegram │ ──────► │ Bot │ ───────► │ Gemini Flash │ │ User │ │ (Zeabur) │ │ (multi-cand) │ └──────────┘ └─────┬─────┘ └──────┬───────┘ ▲ │ candidates[] │ │ msg + map ▼ │ │ ┌─────────────────────┐ │ │ │ Resolver (Cascade) │ │ │ │ │ │ │ │ ① Wikidata API ─────┼──► coords? │ │ ↓ (miss) │ │ │ │ ② Wikipedia API ────┼──► coords? │ │ ↓ (miss) │ │ │ │ ③ Nominatim ────────┼──► coords? │ │ ↓ (miss) │ │ │ │ ④ Photon ───────────┼──► coords? │ └─────────────────────┘ │ │ │ │ └─────────────────────┘ │ formatted reply

處理流程

  1. 使用者傳圖 → Bot 立刻 reply「🔍 識別中…」
  2. Gemini Vision 識別 → 拿到 N 個候選名 + 國家/地區
  3. 解析器級聯:對每個候選依序試 4 個來源,任一命中立即短路
  4. 編輯訊息為:座標 + Google Maps URL
  5. 額外發送 sendLocation 原生地圖訊息

from dataclasses import dataclass, field
from typing import Optional

@dataclass
class PlaceCandidates:
    """Vision 識別結果"""
    candidates: list[str]                    # 1~3 個英文候選名
    place_name_local: Optional[str] = None   # 原文(藏/日/中)
    country: str = ""
    region: str = ""
    description: str = ""
    search_hints: list[str] = field(default_factory=list)
    confidence: str = "low"                  # high|medium|low

@dataclass
class Coords:
    lat: float
    lng: float
    source: str                  # "wikidata"|"wikipedia"|"nominatim"|"photon"
    matched_query: str           # 實際命中的查詢字串
    canonical_name: Optional[str] = None

Prompt(模組常數,完整版)

You are an expert at identifying real-world landmarks from Pikmin Bloom mushroom point screenshots.

ABOUT THE GAME UI:
Pikmin Bloom mushroom screenshots have this layout:
- TOP: Photo of the real-world landmark
- MIDDLE: Landmark name shown as a TITLE (often in multiple scripts)
- BELOW TITLE: A short English description
- "距離" or "Distance": followed by meters — IGNORE THIS
- BOTTOM: 3D mushroom decorations — IGNORE THESE

Your job: identify the landmark from the TITLE and photo. Generate
MULTIPLE candidate names because OCR may produce minor errors.

OUTPUT (valid JSON only, no markdown, no code fence):
{
  "candidates": [
    "Primary English/official name",
    "Alternative spelling or common name",
    "Transliteration variant if applicable"
  ],
  "place_name_local": "Name in original script if visible, else null",
  "country": "Country name in English",
  "region": "City or province in English",
  "description": "One sentence factual description",
  "search_hints": ["extra keyword 1", "extra keyword 2"],
  "confidence": "high" | "medium" | "low"
}

EXAMPLE 1 — Bhutan Stupa Temple:
{
  "candidates": ["Jangtsa Dumtseg Lhakhang", "Dumtseg Lhakhang", "Dungtse Lhakhang"],
  "place_name_local": "ཛྩུ་མ་བར་",
  "country": "Bhutan",
  "region": "Paro",
  "description": "The only temple in Bhutan in the form of a stupa",
  "search_hints": ["Paro chorten", "Thangtong Gyalpo temple"],

}

EXAMPLE 2 — Tokyo Tower:
{
  "candidates": ["Tokyo Tower", "東京タワー"],
  "place_name_local": "東京タワー",
  "country": "Japan",
  "region": "Tokyo",

PHASE THREE · PROVIDER #1
Wikidata Provider

抽象介面 src/providers/base.py

from abc import ABC, abstractmethod
from typing import Optional
from src.models import Coords

class GeocoderProvider(ABC):
    name: str  # "wikidata" 等

    @abstractmethod
    async def lookup(
        self, query: str, hint_country: str = ""
    ) -> Optional[Coords]:
        """單次查詢。找不到回 None,網路錯誤拋 ProviderError。"""

class ProviderError(Exception): ...

Wikidata 三步驟邏輯

  1. Search — 用 wbsearchentities action 拿 Q-id 候選清單(limit=5)
  2. Filter — 篩選 description 含地理關鍵字(temple, tower, mountain, lake, museum, station, palace, monument, park, building, church, mosque, shrine, castle, bridge)的條目;用 hint_country 進一步過濾
  3. Get entity — 對首筆 Q-id 抓 entity JSON,解 P625 取座標

API 範例

# Step 1: Search
GET https://www.wikidata.org/w/api.php
    ?action=wbsearchentities
    &search={query}
    &language=en
    &format=json
    &type=item
    &limit=5

# Step 3: Get entity & coordinates
GET https://www.wikidata.org/wiki/Special:EntityData/{Q-id}.json
# Parse: entities.{Q-id}.claims.P625[0].mainsnak.datavalue.value
# Returns: { "latitude": ..., "longitude": ... }

注意事項

  • 沒 P625(座標屬性)的 entity 視為 miss
  • HTTP timeout 設 8 秒
  • User-Agent 必設:PikminCoordBot/1.0 ({CONTACT_EMAIL})
  • 回傳 Coords(source="wikidata", canonical_name=label)
✓ 驗收條件
await wikidata.lookup("Jangtsa Dumtseg Lhakhang", "Bhutan") 回傳座標約 (27.435, 89.413),誤差 < 0.01 度。

PHASE SEVEN · CASCADE
級聯解析器(核心)

主函式 src/resolver.py

async def resolve(place: PlaceCandidates) -> Optional[Coords]:
    """
    對每個 candidate × 每個 provider 級聯查詢。
    任一命中立即短路。全失敗回 None。
    """

級聯邏輯(行優先:同一 query 先試 4 個 provider)

queries = build_queries(place)
providers = [Wikidata(), Wikipedia(), Nominatim(), Photon()]

for q in queries:
    for p in providers:
        try:
            result = await p.lookup(q, place.country)
            if result:
                logger.info("hit", query=q, provider=p.name)
                return result
        except ProviderError as e:
            logger.warning("provider error", provider=p.name, error=str(e))
            continue
return None

查詢字串生成順序build_queries(place)

  1. f"{candidates[0]}, {region}, {country}"
  2. f"{candidates[0]}, {country}"
  3. f"{candidates[1]}, {country}" — 若有第二候選
  4. f"{candidates[2]}, {country}" — 若有第三候選
  5. f"{place_name_local}, {country}" — 若有原文
  6. f"{candidates[0]}" — 純名字
  7. f"{search_hints[0]}, {country}" — 若有提示

⚠ 去重後再執行,避免重複查詢浪費請求。

✓ 驗收條件
對 PlaceCandidates(candidates=["Jangtsa Dumtseg Lhakhang"], country="Bhutan", region="Paro") 必須回傳座標,且 source == "wikidata"。

handle_photo 完整流程

async def handle_photo(update, context):
    chat_id = update.effective_chat.id
    status = await update.message.reply_text("🔍 識別中…")

    try:
        photo = update.message.photo[-1]
        file = await photo.get_file()
        bio = BytesIO()
        await file.download_to_memory(bio)

        place = await identify_place(bio.getvalue())
        if not place.candidates:
            await status.edit_text(format_unknown())
            return

        await status.edit_text(
            f"🔎 識別到「{place.candidates[0]}」,查詢座標中…"
        )

        coords = await resolve(place)
        if not coords:
            await status.edit_text(
                format_no_coords(place), parse_mode="Markdown"
            )
            return

        await status.edit_text(
            format_success(place, coords),
            parse_mode="Markdown",
            disable_web_page_preview=True,
        )
        await context.bot.send_location(
            chat_id=chat_id,
            latitude=coords.lat,
            longitude=coords.lng,
        )
    except Exception:
        log.exception(
            "handle_photo failed",
            user_id=update.effective_user.id,
        )
        await status.edit_text("⚠️ 處理失敗,請稍後重試")

UX 細節

  • 必須用 edit_message_text 更新「識別中」訊息,不要每步都發新訊息
  • 處理過程超過 5 秒可加 send_chat_action(chat_id, "typing")
  • send_location 要在文字訊息之後送(順序:文字 → 地圖)
  • 任何 exception 都不洩露內容給使用者,只記錄到 log
✓ 驗收條件
本機 Polling 啟動 → Telegram 傳菇點截圖 → 收到三段訊息(識別中 → 查詢中 → 結果)+ 一張原生地圖。

入口程式

from telegram.ext import (
    Application, CommandHandler, MessageHandler, filters,
)
from src.config import settings
from src.bot import cmd_start, handle_photo, handle_non_photo
from src.logger import get_logger

log = get_logger(__name__)

def main():
    log.info("Bot starting", model=settings.LLM_MODEL)
    app = Application.builder().token(settings.TELEGRAM_TOKEN).build()
    app.add_handler(CommandHandler(["start", "help"], cmd_start))
    app.add_handler(MessageHandler(filters.PHOTO, handle_photo))
    app.add_handler(MessageHandler(
        filters.ALL & ~filters.COMMAND, handle_non_photo
    ))
    app.run_polling(allowed_updates=["message"])

if __name__ == "__main__":
    main()

SECTION 10Zeabur 部署步驟

  1. GitHub 上傳 
    git init && git add . && git commit -m "feat: initial bot"
git remote add origin https://github.com/YOUR_USER/pikmin-coord-bot.git
git push -u origin main
  2. 登入 zeabur.com → Create Project
  3. Add Service → GitHub → 選 repo
  4. Zeabur 自動偵測 Python,讀 Procfile 啟動
  5. Variables 頁籤逐一加入 §9 的 3 個必填變數
  6. Deploy,等 build log 出現 Bot starting
  7. 驗證:
    • Telegram 找你的 Bot 傳 /start → 收到歡迎訊息
    • 傳菇點截圖 → 12 秒內收到座標訊息 + 原生地圖
重要:不需要 webhook(用 polling)。不需要對外 port。Service Plan 必須是長駐型,不能用 Serverless,否則 polling 會中斷。
